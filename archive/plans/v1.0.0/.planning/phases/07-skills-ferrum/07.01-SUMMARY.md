# Phase 07 Plan 01: FerrumController Core and Serial Communication Summary

**One-liner:** Serial communication layer with echo-parsing protocol, km.init() initialization, and context manager support for Ferrum hardware device.

---

## What Was Built

### FerrumController Class

A hardware controller class that provides serial communication with Ferrum devices via pyserial:

- **Serial Connection Management**: `_connect()`, `_disconnect()` with proper resource cleanup
- **Command Protocol**: `_send_command()` with `\r\n` terminator, echo handling, and `>>> ` prompt parsing
- **Device Initialization**: `_initialize_device()` sends `km.init()` on connection
- **Connection Validation**: `_validate_connection()` ensures serial port is open before operations
- **Context Manager**: `__enter__`/`__exit__` for automatic cleanup
- **Wait Method**: `wait(seconds)` for ActionDispatcher compatibility

### Protocol Implementation

The Ferrum serial protocol:
- Input: `command\r\n`
- Output: `command\r\n` (echo) + result + `\r\n>>> `
- Response parsing extracts result between echo and prompt

### Error Handling

- `FerrumConnectionError`: Custom exception for connection failures
- Retry-once policy for transient failures
- Timeout handling with monotonic clock
- Proper error logging with `[错误]` prefix

### Logging

Per project convention (Chinese prefix tags):
- `[Ferrum]`: Serial operations, commands, responses (DEBUG/INFO)
- `[错误]`: Error conditions (ERROR level)
