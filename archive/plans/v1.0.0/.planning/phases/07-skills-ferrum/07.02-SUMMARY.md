# Phase 07 Plan 02: Mouse Actions - Move, Click, and Scroll

## Summary

Implemented mouse control methods for FerrumController: `_move()`, `click()`, `click_right()`, and `scroll()`. These methods translate high-level mouse actions into Ferrum KM API commands sent over serial communication.

## One-Liner

Mouse control implementation with relative movement, left/right click, and directional scroll using Ferrum KM API protocol.

## What Was Built

### Core Functionality

1. **`_move(x, y)` helper method**
   - Sends `km.move(x, y)` command for relative mouse movement
   - Documented that coordinates are relative, not absolute
   - Caller must track current position and calculate delta for absolute positioning

2. **`click(x, y)` method**
   - Validates connection before execution
   - Calls `_move(x, y)` to position cursor
   - Sends `km.click(BUTTON_LEFT)` to click left button
   - Logs action with coordinates for debugging

3. **`click_right(x, y)` method**
   - Same pattern as `click()` but uses `BUTTON_RIGHT`
   - Sends `km.click(1)` per Ferrum documentation
   - Available for future use cases requiring right-click

4. **`scroll(direction, ticks)` method**
   - Maps direction: "up" -> +1, "down" -> -1
   - Loops for specified tick count
   - 5ms delay between ticks to avoid overwhelming device
   - Validates direction parameter (must be "up" or "down")

5. **Button Constants**
   - `BUTTON_LEFT = 0`
   - `BUTTON_RIGHT = 1`
   - `BUTTON_MIDDLE = 2`
   - `BUTTON_SIDE_REAR = 3`
   - `BUTTON_SIDE_FRONT = 4`
