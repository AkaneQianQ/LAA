## Key Design Decisions

1. **Relative Movement**: `click()` uses relative coordinates (km.move is relative)
   - Caller must calculate delta from current position to target
   - Documented in integration_example.md

2. **Key Combination Support**: `press()` handles modifiers correctly
   - Parses "alt+u" format
   - Presses modifiers first, releases in reverse order
   - Single keys use km.press(), combinations use km.down()/km.up()

3. **Hardware Detection**: Tests auto-skip if hardware unavailable
   - Allows CI/testing without physical device
   - Mock fallback for interface verification

4. **Context Manager**: Ensures clean resource cleanup
   - `with FerrumController() as ferrum:` pattern
   - Automatic close() on exit

## Usage Example

```python
from core.ferrum_controller import FerrumController
from core.hardware_input_gateway import HardwareInputGateway
from core.workflow_bootstrap import create_workflow_executor

# Create and wrap FerrumController
ferrum = FerrumController(port="COM2")
gateway = HardwareInputGateway(hardware_controller=ferrum)

# Use with workflow system
executor = create_workflow_executor(
    "config/workflows/guild_donation.yaml",
    controller=gateway
)
executor.execute()

# Clean shutdown
ferrum.close()
```
