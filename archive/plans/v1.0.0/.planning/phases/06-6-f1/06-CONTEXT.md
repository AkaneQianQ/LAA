# Phase 6: 交互式测试流程序 - Context

**Gathered:** 2026-03-08
**Status:** Ready for planning

<domain>
## Phase Boundary

创建一个F1驱动的交互式测试界面系统，让用户按照AI指令手动操作游戏并反馈观察结果。这是一个"人工在环"测试系统，用于验证自动化流程的准确性和稳定性。

系统以半透明横向功能区形式覆盖在游戏画面上，显示当前测试指令，接收用户通过快捷键反馈的执行结果。

</domain>

<decisions>
## Implementation Decisions

### UI布局与样式
- **尺寸**: 标准型 (600x80像素)
- **透明度**: 半透明 (70%不透明度)
- **颜色主题**: 深色主题 (深灰/黑色背景，白色文字)
- **拖动区域**: 顶部标题栏 (20像素高)
- **位置**: 默认覆盖在屏幕左上角
- **关闭按钮**: 功能区右侧显示X关闭按钮

### 交互模式
- **指令显示方式**: 覆盖层内文字显示 (功能区中央区域)
- **F1键行为**: 按F1→显示并等待确认 (显示指令后等待用户反馈)
- **用户反馈机制**: 快捷键(Y/N) — Y表示预期达成，N表示异常
- **快捷键定义**:
  - F1: 继续/下一步
  - END: 终止测试
- **注释功能**: 不需要 (简单的通过/失败记录)

### 测试流程管理
- **测试场景定义方式**: Python代码硬编码 (简单直接)
- **测试步骤组织**: 线性顺序 (步骤1→步骤2→步骤3)
- **步骤数量**: 根据项目内需要实现的内容进行步骤设计
- **失败处理**: 终端报错直接弹出，用户按END终止
- **步骤跳过**: 支持 (用户可以跳过当前步骤)
- **场景切换**: 下拉菜单选择 (在功能区显示场景列表)
- **暂停功能**: 不支持 (只能一次性完成或终止)
- **准备确认**: 手动确认 (显示"准备就绪？"，用户按F1确认后开始)

### 数据记录
- **存储格式**: JSON文件 (结构化、易解析)
- **记录详细程度**: 标准记录 (步骤、时间戳、反馈结果)
- **截图策略**: 不截图 (纯文本记录)
- **报告导出**: 不需要 (仅实时查看)

### Claude's Discretion
- 具体JSON格式字段命名
- 下拉菜单的UI实现细节
- 步骤跳过时的状态处理逻辑
- 深色主题的具体颜色值

</decisions>

<specifics>
## Specific Ideas

**工作流程示例**:
1. 用户启动测试程序
2. 功能区显示"准备就绪？"，等待用户按F1
3. 用户按F1后，显示第一步指令如"观察角色选择界面是否出现"
4. 用户执行操作后，按Y(通过)或N(失败)反馈
5. 按F1进入下一步，重复直到完成

**预期指令类型**:
- 观察类: "确认X元素出现在Y位置"
- 操作类: "手动点击Z位置"
- 验证类: "检查A状态是否变为B"

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- **gui_launcher.py**: Tkinter GUI模式，可作为测试界面基础
- **keyboard库**: 全局热键支持 (F1, END等)
- **threading.Event**: 用于测试终止信号

### Established Patterns
- 使用ttk.Frame和ttk.Label进行UI布局
- 全局热键通过keyboard.add_hotkey()注册
- 日志通过scrolledtext.ScrolledText显示
- 中文用户界面消息

### Integration Points
- 测试系统可作为独立模块运行
- 可复用现有的热键管理机制
- 可与VisionEngine集成进行辅助验证

</code_context>
