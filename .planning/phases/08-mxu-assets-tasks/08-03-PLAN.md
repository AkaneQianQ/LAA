  <action>
Create Tauri commands that bridge frontend React to Python WebSocket service.

1. Create frontend/src-tauri/src/commands.rs with:

   a. Python service management:
      - start_python_service(auto_detect: bool) -> Result<ServiceInfo, String>
        * Spawns: python -m agent.py_service.main --server-mode --auto-detect
        * Waits for WebSocket to be ready
        * Returns service info (pid, port, status)
      - stop_python_service() -> Result<(), String>
        * Sends shutdown command via WebSocket
        * Kills process if needed
      - get_service_status() -> ServiceStatus

   b. WebSocket proxy commands:
      - send_websocket_command(action: String, params: Value) -> Result<Value, String>
        * Connects to ws://127.0.0.1:8765
        * Sends command and waits for response
        * Returns parsed JSON response
      - subscribe_to_events() -> Result<Channel<Event>, String>
        * Opens persistent WebSocket connection
        * Streams events to frontend via Tauri Channel

   c. Convenience commands:
      - detect_game_window() -> Result<WindowInfo, String>
      - detect_hardware() -> Result<HardwareInfo, String>
      - recognize_account() -> Result<AccountInfo, String>
      - start_task(task_name: String, options: Value) -> Result<(), String>
      - stop_task() -> Result<(), String>

2. Update frontend/src-tauri/src/lib.rs:
   - Define Tauri builder with all commands
   - Configure permissions for shell execution
   - Set up state management for WebSocket connections

3. Update frontend/src-tauri/src/main.rs:
   - Call the lib run function
   - Add error handling

4. Update frontend/src-tauri/capabilities/default.json:
   - Add shell:allow-execute permission for spawning Python
   - Add shell:allow-spawn permission

Use tauri::State for managing shared WebSocket client state.
Handle Python process lifecycle (spawn, monitor, cleanup).
