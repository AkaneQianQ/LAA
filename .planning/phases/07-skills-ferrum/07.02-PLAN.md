## Implementation Details

### Ferrum KM API Reference

From documentation:

**km.move(x, y)** - Page 26
- x_amount: positive=right, negative=left
- y_amount: positive=down, negative=up
- Units are generic (OS-dependent, typically pixels)
- Both arguments required (use 0 for no movement in axis)

**km.click(button)** - Page 16
- 0 = left button
- 1 = right button
- 2 = middle (scroll wheel)
- Auto press delay: 75-125ms (random)
- Auto release delay: 125-175ms (random)
- Hardware override applies

**km.wheel(amount)** - Page 28
- positive = scroll up
- negative = scroll down
- Security: use only -1 or +1 (other values may look suspicious)

### Method Specifications

**click(x, y)**
1. Move to absolute position: `km.move(x, y)`
2. Click left button: `km.click(0)`
3. Return immediately (delays handled by Ferrum device)

**scroll(direction, ticks)**
1. Determine amount: +1 for "up", -1 for "down"
2. Loop ticks times, sending `km.wheel(amount)` each iteration
3. Brief delay between ticks (5ms) to avoid overwhelming device

<task id="3">
**Implement scroll() method**

Add scroll method:
```python
def scroll(self, direction: str, ticks: int) -> None:
    """
    Scroll the mouse wheel.

    Args:
        direction: 'up' or 'down'
        ticks: Number of scroll ticks
    """
    amount = 1 if direction.lower() == "up" else -1
    for _ in range(ticks):
        self._send_command(f"km.wheel({amount})")
        time.sleep(0.005)  # 5ms between ticks
```

**Verify:** Direction mapping correct (up=+1, down=-1)
</task>
