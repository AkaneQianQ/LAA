## 3. 测试阶段

### Phase 1: 环境检测
- 检测屏幕分辨率 (2560x1440)
- 检测游戏进程

- 打开ESC菜单检测
- 9格角色槽位识别

### Phase 3: 配置系统验证
- 加载YAML工作流配置
- 验证动作类型
- 验证条件分支

### Phase 4: 智能等待验证
- 验证图像等待（出现/消失）
- 超时和重试机制
- 记录实际耗时

### Phase 5: 完整工作流验证
- 执行公会捐赠工作流
- 验证错误恢复
- 记录执行结果

## 5. 技术方案

### 5.1 组件结构

```
tests/acceptance/
├── acceptance_test.py          # 主测试脚本
├── overlay.py                   # 透明覆盖层 (Tkinter)
└── __init__.py
```

### 5.2 核心类

- `OverlayWindow`: Tkinter透明覆盖层，显示状态、进度、日志
- `PhaseRunner`: 阶段执行管理器，与覆盖层交互
- `TestLogger`: 结构化日志记录
- `ScreenshotArchiver`: 截图存档管理

### 5.3 执行流程

```
启动脚本
  ↓
显示覆盖层 (Phase 1/5, 等待F1)
  ↓ (用户按F1)
执行阶段1 → 更新覆盖层 → 等待F1
  ↓ (用户按F1)
执行阶段2 → 更新覆盖层 → 等待F1
  ↓ ...
所有阶段完成
  ↓
生成报告，退出
```
